Goods Receipt POC

Business Problem

Manual GR submission still begins with paper documents and ends with manual keying into a system.

Current pain

  • Delivery Orders and completion certificates are submitted manually
  • PO number, vendor, date, items, and quantities need human review
  • Slow and labour-intensive process

Core User Flow

Capture the document, confirm the fields, submit the GR.

1
Upload DO
User uploads or photographs a Delivery Order in Teams
2
Extract fields
Claude 3.5 Sonnet on Bedrock extracts the delivery date, PO, vendor, items, quantities
3
Pre-fill GR
The app populates the fields with the extracted information, creating an editable draft Goods Receipt
4
Review
User reviews and amends the extracted fields
5
Attach docs
User attaches any supporting documents (e.g. packing list, invoice)
6
Submit
User submits the completed GR with all attachments

System Architecture

Teams hosts the experience. AWS runs extraction, storage, and persistence.

Entry points

Chatbot, low-code app, and full-code tab all call the same backend surface.

Storage and model

DO files use dos/{gr_id}/; attachments use attachments/{gr_id}/. goods_receipt owns line items and attachment records.

Security baseline

Server-side MIME checks, DB secrets in Secrets Manager, and IAM scoped to S3 prefixes.
